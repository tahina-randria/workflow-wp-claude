Setup Initial

Créer le repo GitHub et la structure du projet

0.1 Créer le repo GitHub du projet

Première chose à faire : créer le repo pour tout versionner dès le début.

# Prompt Claude Code :
"Crée un nouveau projet pour la refonte du site [NOM_CLIENT].

1. Crée un dossier /refonte-[nom-client]/
2. Initialise un repo Git
3. Crée la structure de base :
   /refonte-[nom-client]/
   ├── scraping/           # Données extraites du site actuel
   ├── analysis/           # Audits et analyses
   ├── design/             # Design system et assets
   ├── theme-[nom]/        # Thème WordPress
   ├── content-sync/       # Contenu JSON pour sync Git
   ├── docs/               # Documentation client
   ├── CLAUDE_CONTEXT.md   # Contexte pour continuité Claude
   ├── README.md           # Description du projet
   └── .gitignore

4. Crée le repo sur GitHub : [nom-client]-website
5. Push le commit initial

Informations client :
- Nom : [NOM]
- Site actuel : [URL]
- Secteur : [SECTEUR]
- Contact : [EMAIL]"

1.2 Audit SEO complet

Analyser les forces et faiblesses SEO pour savoir quoi améliorer.

# Prompt Claude Code :
"Fais un audit SEO complet du site [URL].

TECHNIQUE (score /100) :
- Vitesse de chargement (PageSpeed)
- Mobile-friendly (responsive)
- HTTPS activé
- Structure des URLs (propres, descriptives)
- Sitemap XML présent
- Robots.txt configuré
- Erreurs 404
- Redirections correctes
- Données structurées (Schema.org)

ON-PAGE (score /100) :
- Balise title (présence, longueur, mots-clés)
- Meta description (présence, longueur, incitation)
- Hiérarchie Hn (H1 unique, H2-H6 logiques)
- Densité mots-clés (naturelle, pas de suroptimisation)
- Maillage interne (liens entre pages)
- Alt text images
- Contenu dupliqué
- Longueur du contenu par page

OFF-PAGE (score /100) :
- Nombre de backlinks estimé
- Qualité des domaines référents
- Présence réseaux sociaux
- Google My Business (si local)
- Mentions de marque

MOTS-CLÉS ACTUELS :
- Sur quels mots le site se positionne ?
- Positions estimées
- Opportunités manquées

GÉNÈRE :
1. Fichier /analysis/audit-seo.md avec scores et détails
2. Liste des problèmes critiques à corriger
3. Liste des améliorations recommandées (priorité haute/moyenne/basse)
4. Mots-clés à cibler pour la refonte"

2.2 Extraire le Design System du site inspiration

Prompt exhaustif pour extraire tous les tokens et styles.

# PROMPT EXHAUSTIF : Extraction Design System

"Extrais le design system complet du site [URL_INSPIRATION].

## 1. PALETTE DE COULEURS
- Couleurs primaires (hex)
- Couleurs secondaires
- Couleurs d'accent
- Couleurs de fond (backgrounds)
- Couleurs de texte (primary, secondary, muted)
- Couleurs d'état (success, warning, error, info)
- Couleurs de bordure
- Gradients utilisés
- Variables CSS si visibles (--color-*)

## 2. TYPOGRAPHIE
- Familles de polices (font-family) + Google Fonts
- Échelle typographique (h1→h6, body, small, caption)
- Font-weight utilisés
- Line-height par niveau
- Letter-spacing
- Styles spéciaux (uppercase, italique)

## 3. ESPACEMENT
- Échelle de margins
- Échelle de paddings
- Gaps (flex/grid)
- Pattern récurrent (multiples de 4px, 8px...)

## 4. LAYOUT
- Largeur max des conteneurs
- Système de grille
- Breakpoints responsive
- Patterns flex/grid utilisés

## 5. COMPOSANTS UI
Détailler pour chaque composant :

BOUTONS :
- Variantes (primary, secondary, outline, ghost)
- Tailles (sm, md, lg)
- États (hover, active, disabled, focus)
- Border-radius, padding, font

FORMULAIRES :
- Style des inputs
- Labels
- Messages d'erreur
- Focus states

CARDS :
- Structure
- Padding
- Shadow
- Border-radius
- Hover effect

NAVIGATION :
- Style navbar
- Menu mobile
- Dropdowns
- Breadcrumbs

AUTRES :
- Badges/tags
- Alertes
- Modales
- Tooltips
- Tableaux

## 6. EFFETS VISUELS
- Border-radius (échelle sm→full)
- Box-shadows (échelle sm→xl)
- Opacités utilisées
- Backdrop-blur

## 7. ANIMATIONS & TRANSITIONS
- Durées (fast, normal, slow)
- Easing functions
- Animations d'entrée/sortie
- Hover effects
- Micro-interactions
- @keyframes si complexes

## 8. ÉTATS INTERACTIFS
- :hover
- :focus / :focus-visible
- :active
- :disabled
- ::selection

## FORMAT DE SORTIE
Génère /design/DESIGN_SYSTEM.md avec :
- Tous les tokens documentés
- Code CSS réutilisable
- Config Tailwind prête à l'emploi"

3.5 Système de sync Git (CMS dynamique)

entre WordPress et GitHub.

- Déclenché après chaque save_post
- Fichiers : services.json, temoignages.json, equipe.json, posts.json

IMPORT JSON → WP :
- Fonction qui importe depuis les fichiers JSON

WORKFLOW :
1. Client modifie dans WP → Export auto en JSON
2. Dev modifie JSON via Claude → Import au deploy
3. Tout est versionné dans Git

GITHUB ACTIONS :
- Déclenché sur push branch main
- Build assets (npm run build)
- Vérif syntaxe PHP
- Deploy FTP/SSH vers serveur

5.1 Configurer GitHub Actions 

# .github/workflows/deploy.yml

name: Deploy WordPress Theme

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install & Build
        run: |
          cd theme-[nom]
          npm ci
          npm run build

      - name: Verify PHP
        run: |
          for f in $(find theme-[nom] -name "*.php"); do
            php -l "$f" || exit 1
          done

      - name: Deploy FTP
        uses: SamKirkland/FTP-Deploy-Action@v4.3.4
        with:
          server: ${{ secrets.FTP_SERVER }}
          username: ${{ secrets.FTP_USERNAME }}
          password: ${{ secrets.FTP_PASSWORD }}
          local-dir: ./theme-[nom]/
          server-dir: /www/wp-content/themes/theme-[nom]/
          exclude: |
            **/.git*/**
            **/node_modules/**
            package*.json
            vite.config.js
            tailwind.config.js

      - name: Notify
        run: echo "✅ Deployed!"
