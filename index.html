Phase 2 : Design System

2.1 Extraction de Design System depuis HTML/CSS

Prompt exhaustif pour extraire un design system complet d'un site existant.

# PROMPT : Extraction de Design System depuis HTML/CSS

## OBJECTIF
Extraire un système de design exhaustif, généralisé et
réutilisable à partir des fichiers HTML et CSS fournis,
en isolant exclusivement les règles stylistiques et
structurelles.

## ENTRÉE ATTENDUE
Fichiers HTML et/ou CSS (code source, liens, ou contenu collé).

## TÂCHE - Analyser et documenter :

### 1. Palette de couleurs
- Couleurs primaires, secondaires, accent
- Couleurs de fond (backgrounds)
- Couleurs de texte (primary, secondary, muted, disabled)
- Couleurs d'état (success, warning, error, info)
- Couleurs de bordure
- Gradients éventuels
- Variables CSS custom properties (--color-*)

### 2. Typographie
- Familles de polices (font-family) et fallbacks
- Échelle typographique (font-size : h1→h6, body, caption)
- Graisses (font-weight)
- Hauteurs de ligne (line-height)
- Espacement des lettres (letter-spacing)
- Styles de texte (italique, uppercase, etc.)

### 3. Espacement (Spacing)
- Échelle de marges (margin)
- Échelle de paddings
- Gaps (flexbox/grid)
- Variables ou patterns récurrents (multiples de 4px, 8px)

### 4. Mise en page (Layout)
- Système de grille (colonnes, gutters)
- Conteneurs (max-width, padding latéral)
- Breakpoints responsive
- Règles Flexbox/Grid utilisées
- Patterns de disposition (stack, cluster, sidebar)

### 5. Composants UI
- Boutons : variantes, tailles, états (hover, active, disabled)
- Champs de formulaire : inputs, selects, textareas
- Cartes : structure, padding, ombres
- Modales/Dialogs
- Navigation : navbar, sidebar, tabs, breadcrumbs
- Tableaux : styles de lignes, headers, bordures
- Badges, tags, chips
- Alertes/Notifications
- Tooltips, popovers

### 6. Effets visuels statiques
- Border-radius (échelle : sm, md, lg, full)
- Box-shadows (échelle : sm, md, lg, xl)
- Opacités
- Filtres (blur, backdrop-filter)
- Outlines
- Text-shadows

### 7. Animations & Transitions

#### 7.1 Transitions
- Propriétés animées (transform, opacity, color, background)
- Durées (transition-duration)
- Fonctions de timing (ease, ease-in-out, cubic-bezier)
- Délais (transition-delay)
- Patterns récurrents par composant

#### 7.2 Keyframes Animations
- Noms des animations (@keyframes)
- Étapes clés (0%, 50%, 100%, from, to)
- Propriétés animées à chaque étape
- Durée (animation-duration)
- Fonction de timing
- Nombre d'itérations
- Direction (normal, reverse, alternate)
- Mode de remplissage (forwards, backwards, both)

#### 7.3 Types d'animations identifiées
- Entrées : fade-in, slide-in, scale-in
- Sorties : fade-out, slide-out, scale-out
- Attention/Feedback : shake, pulse, bounce, wiggle
- Chargement : spin, skeleton, shimmer, progress
- Micro-interactions : hover lifts, button press
- Scroll-based : parallax, reveal on scroll

### 8. États et interactions
- Styles :hover
- Styles :focus / :focus-visible
- Styles :active
- Styles :disabled
- Styles ::selection
- Styles :checked, :indeterminate
- Styles :valid, :invalid

### 9. Tokens & Variables
- Liste complète des CSS custom properties
- Patterns de nommage identifiés
- Organisation par catégorie

## FORMAT DE SORTIE
- Valeurs exactes extraites du code
- Organisation par catégorie
- Exemples de code CSS réutilisables
- Tableau récapitulatif des tokens
- Code des @keyframes complet

## CONTRAINTES
- NE PAS extraire le contenu textuel, labels, libellés
- NE PAS extraire les URLs d'images, logos spécifiques
- NE PAS inclure les classes utilitaires liées au contenu
- SE CONCENTRER sur les règles de style abstraites
- ÊTRE EXHAUSTIF : ne rien omettre

2 GitHub Actions - Deploy WordPress 

# .github/workflows/deploy.yml

name: Deploy WordPress Theme

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build assets
        run: npm run build

      - name: Verify PHP syntax
        run: |
          for f in $(find . -name "*.php"); do
            php -l "$f"
          done

      - name: Deploy via FTP
        uses: SamKirkland/FTP-Deploy-Action@v4.3.4
        with:
          server: ${{ secrets.FTP_SERVER }}
          username: ${{ secrets.FTP_USERNAME }}
          password: ${{ secrets.FTP_PASSWORD }}
          local-dir: ./
          server-dir: /www/wp-content/themes/theme-name/
          exclude: |
            **/.git*
            **/.git*/**
            **/node_modules/**
            .env
            package*.json
            vite.config.js
            tailwind.config.js
            postcss.config.js

      - name: Notify success
        run: echo "✅ Theme deployed successfully!"

3 Sync Contenu WP → GitHub

Exporter le contenu WordPress en JSON pour versionner dans Git.

<?php
// inc/export-content.php

/**
 * Exporte tout le contenu en JSON pour sync Git
 */
function export_content_to_json() {
    $export_dir = URTAM_DIR . '/content-sync/';

    if (!file_exists($export_dir)) {
        mkdir($export_dir, 0755, true);
    }

    // Exporter chaque CPT
    $post_types = ['service', 'temoignage', 'membre_equipe', 'post', 'page'];

    foreach ($post_types as $post_type) {
        $posts = get_posts([
            'post_type' => $post_type,
            'numberposts' => -1,
            'post_status' => 'publish',
        ]);

        $data = [];
        foreach ($posts as $post) {
            $data[] = [
                'id' => $post->ID,
                'title' => $post->post_title,
                'slug' => $post->post_name,
                'content' => $post->post_content,
                'excerpt' => $post->post_excerpt,
                'date' => $post->post_date,
                'meta' => get_post_meta($post->ID),
                'thumbnail' => get_the_post_thumbnail_url($post->ID, 'full'),
            ];
        }

        file_put_contents(
            $export_dir . $post_type . '.json',
            json_encode($data, JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE)
        );
    }

    return true;
}

// Hook sur save_post pour auto-sync
add_action('save_post', function($post_id) {
    if (defined('DOING_AUTOSAVE') && DOING_AUTOSAVE) return;
    export_content_to_json();
}, 10, 1);

// Bouton manuel dans admin
add_action('admin_post_export_content_git', function() {
    export_content_to_json();

    // Optionnel: commit + push automatique
    $output = shell_exec('cd ' . URTAM_DIR . ' && git add content-sync/ && git commit -m "Auto-sync content" && git push 2>&1');

    wp_redirect(admin_url('admin.php?page=dashboard&exported=1'));
    exit;
});

5 Import JSON → WordPress (bidirectionnel) 

<?php
// inc/import-from-json.php

/**
 * Importe le contenu depuis les fichiers JSON
 * (après un git pull ou deploy)
 */
function import_content_from_json() {
    $import_dir = URTAM_DIR . '/content-sync/';
    $post_types = ['service', 'temoignage', 'membre_equipe'];

    foreach ($post_types as $post_type) {
        $file = $import_dir . $post_type . '.json';
        if (!file_exists($file)) continue;

        $data = json_decode(file_get_contents($file), true);

        foreach ($data as $item) {
            // Chercher si existe déjà (par slug)
            $existing = get_page_by_path($item['slug'], OBJECT, $post_type);

            $post_data = [
                'post_title' => $item['title'],
                'post_name' => $item['slug'],
                'post_content' => $item['content'],
                'post_excerpt' => $item['excerpt'] ?? '',
                'post_type' => $post_type,
                'post_status' => 'publish',
            ];

            if ($existing) {
                $post_data['ID'] = $existing->ID;
                wp_update_post($post_data);
            } else {
                $post_id = wp_insert_post($post_data);
            }

            // Mettre à jour les meta
            if (!empty($item['meta'])) {
                foreach ($item['meta'] as $key => $value) {
                    update_post_meta(
                        $existing ? $existing->ID : $post_id,
                        $key,
                        is_array($value) ? $value[0] : $value
                    );
                }
            }
        }
    }

    return true;
}

// Hook après git pull (via webhook ou cron)
add_action('wp_loaded', function() {
    if (isset($_GET['sync_from_git']) && current_user_can('manage_options')) {
        shell_exec('cd ' . URTAM_DIR . ' && git pull');
        import_content_from_json();
        wp_redirect(admin_url('admin.php?page=dashboard&synced=1'));
        exit;
    }
});
